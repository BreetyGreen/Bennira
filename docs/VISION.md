# Bennira 产品愿景

## 一句话方向

Bennira 想成为一个面向个人开发者的本地代码 Agent：它能理解代码库、和用户讨论需求、调用工具完成小步修改，并把每一次行动解释清楚。

## 为什么做

Codex 和 Claude Code 这类产品厉害，不只是因为模型强，也因为它们把模型接入了真实开发环境：

- 能读取和搜索代码库。
- 能修改文件并展示差异。
- 能运行命令、测试、lint 和构建。
- 能根据错误继续修正。
- 能遵守项目规则和权限边界。
- 能把复杂任务拆成可执行的小步骤。

Bennira 的探索目标，是理解并实现这套能力的核心骨架。

## 长期能力想象

- 代码库理解：识别项目结构、技术栈、入口文件、测试方式。
- 对话式开发：把模糊想法拆成需求、设计和任务。
- 工具调用：读文件、搜代码、改文件、跑命令、看 Git 状态。
- 安全执行：区分读、写、执行、联网、删除、提交等权限。
- 插件系统：把文件、Git、Shell、浏览器、文档、MCP 等能力模块化。
- 记忆与规范：读取项目规则，并在后续任务中持续遵守。
- 验证闭环：修改后运行验证，失败时解释原因并继续修。

## 现阶段判断

现在还不需要马上决定完整架构，也不需要马上做复杂 UI。

更重要的是先完成三件事：

1. 把想法写清楚。
2. 把“小偷”版本的 Alpha 与 MVP 边界定义清楚。
3. 确保换电脑后项目可以继续。

## “小偷”版本的产品隐喻

“小偷”不是指破坏，而是指轻量、谨慎、会观察。

这个版本应该先学会：

- 悄悄进入一个代码库。
- 观察目录和文件。
- 找到关键线索。
- 做小而准确的改动。
- 留下清楚的行动记录。
